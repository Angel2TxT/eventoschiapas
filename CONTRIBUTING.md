# Guía de colaboración

Este proyecto está pensado como práctica académica. Si otra persona colabora, conviene seguir estas reglas:

## Flujo sugerido

1. Crear una rama por tarea.
2. Hacer cambios pequeños y claros.
3. Revisar antes de subir.
4. Abrir un Pull Request hacia `main`.
5. Resolver comentarios y conflictos con cuidado.

## Nombres de ramas

Usa nombres descriptivos, por ejemplo:

- `rama-header`
- `rama-footer`
- `rama-contacto`
- `rama-eventos`

## Mensajes de commit

Escribe mensajes breves y explicativos, por ejemplo:

- `Estructura inicial del sitio web`
- `Agregar sección de eventos`
- `Mejorar diseño responsive`
- `Documentar conflicto resuelto`

## Reglas básicas

- No mezclar muchos cambios en un solo commit.
- Mantener el código legible.
- Revisar que el sitio siga funcionando antes de hacer merge.
- Documentar cualquier conflicto en `CONFLICT-LOG.md`.
